# 多模态学生模型多层次知识蒸馏机制消融实验设计 (ECG-PCG)

## 1. 实验目的
本实验旨在定量剖析并证明在异构多模态网络（CNN 教师网络指导 Transformer 学生网络）中，**逻辑值蒸馏（Logits KD）**与**中间层特征对齐蒸馏（Feature KD）**对学生模型（ECG+PCG）最终分类性能、收敛速度以及模态对齐能力的各自独立贡献与协同效应。

---

## 2. 核心损失函数与模块定义
学生模型的总体优化目标由三部分损失加权构成：

$$ \mathcal{L}_{total} = \alpha \mathcal{L}_{hard} + \beta \mathcal{L}_{soft} + \gamma \mathcal{L}_{feat} $$

* **$\mathcal{L}_{hard}$ (硬标签交叉熵):** 基于 Ground Truth 的基础监督信号。
* **$\mathcal{L}_{soft}$ (逻辑值软标签):** 约束师生模型最终决策层（Softmax概率分布）的 KL 散度，传递类间关系的“暗知识”。
* **$\mathcal{L}_{feat}$ (对齐层特征损失):** 约束师生模型在多模态对齐层（Alignment Layer）输出特征的均方误差（MSE），传递局部特征提取与跨模态融合的“空间知识”。

---

## 3. 实验分组方案设计 (Experimental Grouping)

为了全面拆解蒸馏机制的作用，实验分为三个递进阶段：**基础组件消融**、**模态贡献消融**以及**对齐阶段消融**。

### 第一阶段：基础蒸馏组件消融 (核心验证)
**目的：** 验证软标签和特征对齐的独立价值及叠加效应。

| 实验编号 | 分组名称 (Group Name) | $\alpha$ | $\beta$ (Soft) | $\gamma$ (Feat) | 核心验证目的与预期假设 |
| :---: | :--- | :---: | :---: | :---: | :--- |
| **Exp 1** | **Baseline (Student Only)** | 1.0 | 0 | 0 | **基线对比。** 评估纯 Transformer 学生模型在无指导情况下的天然表现（预期收敛慢、极易过拟合）。 |
| **Exp 2** | **Student + Logits KD** | 1.0 | 1.0 | 0 | **决策层指导。** 验证仅模仿老师最终预测概率的效果（预期泛化性有一定提升，但多模态对齐依然困难）。 |
| **Exp 3** | **Student + Feature KD** | 1.0 | 0 | 1.0 | **对齐层指导。** 验证 CNN 老师直接指导 Transformer 学生提取生理信号特征的作用（预期收敛极快，性能大幅提升）。 |
| **Exp 4** | **Student + Full KD** | 1.0 | 1.0 | 1.0 | **全路径蒸馏。** 验证特征级与决策级蒸馏的协同效应（预期达到全局最优性能）。 |

### 第二阶段：模态级特征蒸馏消融 (细粒度分析)
**目的：** 探讨在 $\mathcal{L}_{feat}$ 中，是 ECG 特征更需要老师指导，还是 PCG 特征更需要老师指导。

| 实验编号 | 分组名称 | $\gamma_{ecg}$ | $\gamma_{pcg}$ | 核心验证目的与预期假设 |
| :---: | :--- | :---: | :---: | :--- |
| **Exp 5** | **Feature KD (ECG Only)** | 1.0 | 0 | 仅蒸馏心电对齐特征。探究模型对 ECG 模态的依赖度。 |
| **Exp 6** | **Feature KD (PCG Only)** | 0 | 1.0 | 仅蒸馏心音对齐特征。由于心音通常噪声更大，预期指导 PCG 的边际收益更高。 |

### 第三阶段：对齐机制空间消融 (深度机理分析)
**目的：** 证明在“对齐后（Post-alignment）”进行特征蒸馏，优于在“对齐前（Pre-alignment，即独立 Encoder 输出端）”进行蒸馏。

* **Exp 7 (Pre-Alignment KD):** 计算损失的位置位于 `ecg_encoder` / `pcg_encoder` 的输出，尚未进入 `AlignmentLayer`。
* **Exp 8 (Post-Alignment KD):** (等同于 Exp 3) 计算损失的位置位于 `AlignmentLayer` 融合之后的特征。
* **对比价值：** 证明老师传授的最具价值的知识，不是单纯的单模态特征，而是“如何将 ECG 和 PCG 映射到统一特征空间”的复合能力。

---

## 4. 丰富的可视化与分析方案 (Visualization & Analysis)

### 4.1 学习动力学分析：收敛曲线图 (Convergence Dynamics)
* **绘制内容：** 将 Exp 1 (Baseline), Exp 2 (Logits), Exp 3 (Feature), Exp 4 (Full) 的 `Validation Loss` 和 `Validation F1-Score` 随 Epoch 变化的曲线绘制在同一张图上。
* **论证逻辑：** Transformer 模型（学生）缺乏归纳偏置（Inductive Bias），从头训练 1D 信号通常在早期陷入震荡。预期通过曲线可清晰展示：**加入了 Feature KD (Exp 3/4) 的模型，在前 10-20 个 Epoch 会出现极其陡峭的性能爬升**，直观证明了 CNN 老师的特征空间指导大大加速了学生的特征学习过程。

### 4.2 模态对齐质量分析：t-SNE/UMAP 特征流形可视化
* **绘制内容：** 抽取验证集数据，分别提取 Exp 1 (无特征蒸馏) 和 Exp 3 (有特征蒸馏) 中 `AlignmentLayer` 输出的 `ecg_feat_aligned` 和 `pcg_feat_aligned`。使用 t-SNE 降维至 2D 平面绘制散点图。
  * *颜色编码 (Color):* 按疾病类别分类（如：正常=蓝，早搏=红）。
  * *形状编码 (Shape):* 按模态分类（如：ECG=圆形，PCG=三角形）。
* **论证逻辑：**
  * **Exp 1 (无蒸馏) 的图：** 预期圆形（ECG）和三角形（PCG）会产生明显的“模态簇（Modality Gap）”，即同属一类疾病两者的特征也相距较远，说明学生未真正融合它们。
  * **Exp 3 (有蒸馏) 的图：** 预期相同颜色的圆形和三角形会高度重合、紧密聚类。这强有力地证明了中间层特征蒸馏教会了学生“打破异构信号的模态壁垒”。

### 4.3 “暗知识”传递分析：Softmax 概率分布热力图
* **绘制内容：** 挑选若干“困难样本（Hard Samples）”（例如：心电图轻微异常，但心音杂音明显）。展示其 Ground Truth、Teacher、Exp 1 (无软标签) 和 Exp 2 (有软标签) 的输出概率柱状图/热力图。
* **论证逻辑：** Ground Truth 是绝对的 `[0, 1, 0, 0]`（硬标签）。Teacher 可能输出 `[0.1, 0.7, 0.2, 0.0]`。预期展示：引入 Logits KD 后，学生不再盲目给出非黑即白的预测，而是学会了次优概率（Secondary Probabilities），证明软标签平滑了分类边界并提升了泛化性。

### 4.4 模态鲁棒性测试：模态缺失雷达图 (Modality Masking Radar Chart)
* **绘制内容：** 在测试集推理时，人为施加干扰：
  1. 完整输入 (ECG + PCG)
  2. PCG 静音/加噪 (ECG + Noise)
  3. ECG 屏蔽/加噪 (Noise + PCG)
* **论证逻辑：** 记录各组模型在上述三种情况下的表现，绘制多边形雷达图。多模态小模型极易发生“模态坍塌（仅依赖强模态）”。预期展示：经过 Full KD (Exp 4) 的模型，在缺失单一模态时性能下降最少，证明特征蒸馏帮助模型在联合表示空间中保留了双模态的完整互补信息。


# 多模态诊断模型与 LLM 结合架构方案 (Scheme A & B)

## 1. 方案背景
本方案旨在解决判别式深度学习模型（ECG/PCG 神经网络）在临床应用中“黑盒化”的问题。通过引入大语言模型（LLM），将多模态特征空间映射到自然语言空间，从而实现**增强的诊断正确性**与**医学可解释性**。

---

## 2. 方案 A：语义映射辅助诊断 (Prompt Engineering Path)
此方案侧重于将模型生成的判别结果转化为 LLM 可理解的结构化临床描述。

### 2.1 核心流程
1. **结构化提取**：从多模态模型输出层获取类别概率、异常得分，并结合位置元数据（loc）进行量化。
2. **知识模版化**：将模型提取的特征（如 ECG 的心律变异性、PCG 的第一心音增强等）转化为医学描述文本。
3. **提示词工程 (Prompting)**：
    - **背景输入**：患者年龄、病史、症状描述。
    - **模型发现**：多模态模型识别出的体征（如：ECG 识别出 ST 段改变，PCG 识别出舒张期杂音）。
    - **推理任务**：要求 LLM 根据医学文献验证模型发现的合理性，并给出逻辑解释。

### 2.2 逻辑架构
- **Input**: Multimodal Model Outputs (Probs, Stats) + Meta Data.
- **Processing**: Semantic Mapping Layer.
- **Model**: LLM (OpenAI, Med-Llama, etc.) as a Reasoning Engine.
- **Output**: Explanatory Diagnostic Report.

---

## 3. 方案 B：特征嵌入空间对齐 (Feature Embedding Alignment)
此方案参考 LLaVA 或 BLIP-2 架构，将多模态模型的 Embedding 空间与 LLM 的词向量空间进行数学对齐。

### 3.1 核心流程
1. **特征提取器 (Backbone)**：冻结已训练好的 ECG 和 PCG 编码器，提取高维语义向量。
2. **线性投影层 (Projection Layer)**：设计一个轻量级的线性转换层（或小规模 MLP），将判别模型的特征维度映射到 LLM 的隐藏层维度。
3. **联合 Embedding 输入**：将映射后的“信号特征向量”作为 LLM 的虚拟 Token（类似于图像 Prompt），与文本指令一同输入 LLM。
4. **端到端推理**：LLM 直接感知信号特征，而非文字结果，从而捕捉判别结果中无法描述的微小波形变化。

### 3.2 逻辑架构
- **Input**: Raw Signal Embedding (ecg_feat, pcg_feat).
- **Connector**: Trainable Adapter (Linear/MLP).
- **Model**: Frozen LLM acting as a Multi-modal Decoder.
- **Output**: Signal-to-Language Generative Reasoning.

---

## 4. 方案对比分析

| 特性 | 方案 A：语义映射 | 方案 B：特征对齐 |
| :--- | :--- | :--- |
| **集成深度** | 浅层：逻辑层面集成 | 深层：特征空间集成 |
| **开发成本** | 低：无需重新训练模型 | 高：需微调投影层 |
| **解释能力** | 基于预设标签的文本解释 | 基于全量特征的生成解释 |
| **泛化性** | 强：可适配任何 LLM API | 中：需特定训练对齐层 |
| **医疗纠错** | 依赖 LLM 的逻辑常识校正 | 具备修正模型分类错误的能力 |

---

## 5. 增强诊断正确性与可解释性的逻辑

### 5.1 正确性增强
- **交叉验证**：LLM 可以识别模型结果与临床常识之间的冲突（例如：ECG 显示极高心率但 PCG 显示正常跳动）。
- **长文本上下文**：引入模型无法感知的电子病历（EMR）信息，补全诊断拼图。

### 5.2 可解释性增强
- **因果推理**：LLM 能将波形变化（如 ST 段抬高）与病理机制（如心肌缺血）挂钩，解释“为什么”得出结论。
- **动态交互**：允许医生通过对话提问（如：“为什么模型认为 PCG 信号中有杂音？”），LLM 溯源特征向量给出说明。

---

## 6. 实施建议
1. **初期阶段**：采用方案 A。利用模型现有的 Softmax 分数和标签，快速建立诊断报告生成系统。
2. **进阶阶段**：采用方案 B。训练一个专用的 Adapter 层，将 ECG/PCG 特征送入私有化部署的医学大模型，实现真正的多模态语义理解。
